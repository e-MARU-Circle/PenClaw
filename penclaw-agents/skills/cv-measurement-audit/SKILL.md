# CV計測棚卸し（コンバージョン計測 監査・再構築）

歯科医院のGoogle広告で「コンバージョン数は多いのに相談が増えない」状態を、偽CVの特定→停止→正計測の構築で解消する手順。担当はハブ（SE）＋マコト（Webマーケ）、統括はカイ。

## いつ使うか
- Adsのコンバージョン数がKPI（相談件数）と乖離している
- 何をCVに数えているか分からない／GBPや閲覧がCVに混ざっている疑い
- gclidオフライン取込・拡張コンバージョンを設計/修理したい

## 行動境界（必ず守る）
- **同意チェック（顧客データ共有ポリシー）・共有権限の変更は代行しない**。先生のクリックを得る（規約同意は明示許可があれば代行可、共有権限変更は不可）。
- **患者の個人情報（メール・電話）をGoogleに送らない**のが既定方針＝拡張コンバージョンは原則OFF、gclidのみ。変更時は必ず先生に確認。
- Ads/サイトへの実書き込みは先生のGO後。

## 運用前提（江間ファミリー歯科）
- Google Ads customer_id: `510-364-0388`（MCPツールで操作。get/list系は[object Object]で落ちることがある→performance系・ブラウザ操作で代替）
- GA4: プロパティ `420948539` / 測定ID `G-XN8D9MQCRX`
- 取込シート「Ads_CV取込_江間ファミリー歯科」ID `1XKPRb3_zCzPJABOaBkbJHYe-djox2sV9CbMa8yHnBZg`（タブ: click_conversions=gclid / enhanced_leads=PII）
- フォーム: 患者お問合せ（Googleフォーム）。送信時Apps Scriptが GA4 generate_lead 送信＋シート出力。gclidプレフィル entry: `entry.1213300835`
- LP3本（/implant/ /orthodontics/ /kids/）は自己完結テンプレ。標準ページは footer.php を読む。

## 手順

### STEP1. 現状把握（読み取り）
1. `google_ads_account_performance` / `google_ads_campaign_performance`（直近30日）でクリック・費用・conversions を取得。KPI（相談）と比較し水増し率を見る。
2. **コンバージョンアクション棚卸し**: Ads管理画面 目標>概要>すべてのコンバージョンアクション。各アクションの「アクションの最適化（メイン/サブ）」「目標カテゴリ」「件数」を一覧化。
3. `ga4_list_conversion_events` ＋ `ga4_run_report`（dimensions: eventName, pagePath / metric: eventCount）で、どのイベントがどのページで発火しているか確認。

### STEP2. 診断
- 偽CVの典型: ページ閲覧（例 /first/ の `ads_conversion___1`）、GBPローカルアクション（経路・サイト訪問・その他エンゲージ・通話ボタン**クリック**）、フォーム**クリック**。これらは「相談」ではない。
- **GBP（Googleホスティング）由来CVは個別にメイン⇔サブ変更不可＝目標カテゴリ単位で管理**。コンバージョンアクション設定画面に最適化編集UIは出ない。
- 入札方式を確認（キャンペーン設定）。コンバージョン自動入札なら、メイン目標を変えると学習がリセット（1〜2週変動）する点を先生に共有。

### STEP3. 偽CVを入札から外す（Phase0・止血）
- 推奨手段＝**各キャンペーンのコンバージョン目標を「アカウントのデフォルト」→「キャンペーン固有」に切替え、実相談だけ選択**（=「予約」＋「見込み顧客からの電話」）。連絡先/ルート/離脱/その他/ページビュー/エンゲージは除外。
- リバーシブルで安全。アクション個別降格より確実（GBPは個別不可のため）。

### STEP4. gclid捕捉の検証・修理（Phase1）
- 各広告LPに `?gclid=TEST` で着地→JSで `a[href*="docs.google.com/forms"]` のhrefに `entry.1213300835=<値>` が入るか確認（送信不要）。cookie名は `gclid`。
- メイン予約ページ /reservation/ が漏れていたら footer.php 末尾に「gclid cookie読取→フォームリンクにprefill」スニペットを追記（冪等・バックスラッシュ不使用）。テーマエディタはCodeMirror `setValue`/`replaceRange`経由（textarea直書きは巻き戻る）。WAFは通常GETページ表示には非該当。

### STEP5. オフライン取込の構築（Phase2/3）
1. 目標>概要>＋コンバージョンアクション作成 > データソース＝**オフライン コンバージョン**のみ > 「後でデータソース設定」 > **顧客データ同意（先生のクリック）** > 完了。
2. カテゴリは入札対象の目標に合わせる（例: **予約**）。名前は **Apps ScriptのCONVERSION_NAMEと完全一致**（例: `相談_フォーム`）。メインに設定。
3. 目標>アップロード>スケジュール > Googleスプレッドシート > 取込シートをリンク > 頻度=24時間ごと。**先頭タブを click_conversions（gclid列）にする**（enhanced_leadsが先頭だと列不一致でエラー＝患者PII回避にもなる）。
4. **先生がシートをサービスアカウントに共有**（閲覧可）: `897424758-100000002-account@partnercontent.gserviceaccount.com`。未共有だと「ファイルが見つかりません」。

### STEP6. 既知のエラーと修正
- **「Conversion Timeが無効」**: タイムゾーンは**コロン不可**。`+09:00`→`+0900`。Apps Scriptの `formatDate(...)+'+0900'`（または `"...HH:mm:ssZ"`）に修正。
- ダミーgclid（テスト行）は常にエラー＝正常。最終確認は実広告クリック→LP→送信→翌取込で受理＞0。
- 拡張コンバージョン(enhanced_leads)は既定OFF。使う場合のみ先生の明示承認＋PIIタブを別管理。
